## Overview

The pipeline has three stages:

1. **Data preparation** (`mcndroid_monthwise_split_and_loader.py`) — splits the master metadata CSV into one CSV per calendar month so both adaptation scripts can iterate month-by-month.
2. **CADE adaptation** (`mcndroid_cade_budget_adaptation_monthwise.py`) — trains a Contrastive Autoencoder on 2013 data, then adapts it month-by-month with a limited labelling budget.
3. **Chen et al. (2023) adaptation** (`mcndroid_chen_budget_adaptation_monthwise.py`) — implements the Hierarchical Contrastive Learning baseline from Chen et al. (USENIX Security 2023) on the same monthwise setup for comparison.

For each month the CADE script:
- Detects concept drift using MAD-based anomaly scoring in the latent space.
- Selects up to `budget` samples for adaptation (using one of three selection policies).
- Fine-tunes the CADE and retrains the classifier on the accumulated memory.
- Evaluates on the held-out samples for that month.

Results are reported across three temporal regimes — **IID** (2014), **Near** (2016–2017), and **Far** (2018–2025).

## Data Layout

The scripts expect data to be placed under the following paths relative to the script location. Training year is fixed at **2013**. Test years are **2014, 2016–2025** (2015 is skipped).

### Master metadata CSV (required for Step 1)

```
final_hash_date_label_family.csv (https://zenodo.org/records/19969833 -> Download metadata.csv)
```

Must contain these columns:

| Column | Description |
|--------|-------------|
| `hash` | SHA-256 hash of the APK |
| `date` | Date the sample was collected (any pandas-parseable format) |
| `label` | `1` = malware, `0` = benign |
| `family` | Malware family name (or benign label) |

### Static modality feature files

```
data-features/init_2013/
└── <year>/
    ├── train_X.npz       # Sparse feature matrix
    ├── train_meta.npz    # Labels (y) and APK hashes
    ├── test_X.npz
    └── test_meta.npz
```

### Graph modality feature files

```
gml-features/init_2013/
└── <year>/
    ├── train_X_y.npz     
    └── test_X_y.npz
```

### Dynamic modality feature files

```
json-features/init_2013/
└── <year>/<year>/
    ├── train_X.npz
    ├── train_meta.npz
    ├── test_X.npz
    └── test_meta.npz
```

### All (fusion) modality

Requires all three feature sets above to be present simultaneously. Samples are aligned across modalities by APK hash — only hashes present in all three are used.

## Step 1 — Prepare Monthwise Data

> **This step must be completed before running the next two steps** 

### What the data loader does

`mcndroid_monthwise_split_and_loader.py` reads the master metadata CSV (`final_hash_date_label_family.csv`) and:

1. Parses the `date` column and derives a `YYYY-MM` month string for every sample.
2. Writes one CSV per month under `monthwise_data/data/monthwise_csv/<year>/<YYYY-MM>.csv`.
3. Writes a `month_summary.csv` with per-month sample counts for a quick sanity check.

### Running the data loader

The `__main__` block has two hardcoded paths:

```python
csv_path = "final_hash_date_label_family.csv"
out_dir  = "monthwise_csv"
```

This means running it bare:

```bash
python3 mcndroid_monthwise_split_and_loader.py
```

will only work if `final_hash_date_label_family.csv` is in the current directory, and it will write output to `monthwise_csv/` — **not** the `monthwise_data/data/monthwise_csv/` path that the CADE and Chen scripts expect.

**The recommended approach** is to call `split_csv_monthwise` directly with the correct paths, either in a short Python snippet or a setup script:

```python
from mcndroid_monthwise_split_and_loader import split_csv_monthwise

summary = split_csv_monthwise(
    csv_path="final_hash_date_label_family.csv",
    out_dir="monthwise_data/data/monthwise_csv",
)
print(summary.head(12).to_string(index=False))
```

Alternatively, edit the two hardcoded lines at the bottom of `mcndroid_monthwise_split_and_loader.py` before running it:

```python
if __name__ == "__main__":
    csv_path = "final_hash_date_label_family.csv"       # ← path to your CSV
    out_dir  = "monthwise_data/data/monthwise_csv"      # ← change this line
    ...
```

Then run it normally:

```bash
python3 mcndroid_monthwise_split_and_loader.py
```

### Using the loader programmatically

The script exposes three helper functions you can import in your own notebooks or pipelines:

**`split_csv_monthwise(csv_path, out_dir)`** — splits the master CSV into per-month files and returns a summary DataFrame.

```python
from mcndroid_monthwise_split_and_loader import split_csv_monthwise

summary = split_csv_monthwise(
    csv_path="final_hash_date_label_family.csv",
    out_dir="monthwise_data/data/monthwise_csv",
)
print(summary.head(12))
```

**`build_hash_to_month_map(csv_path)`** — returns a `{sha256_hash: 'YYYY-MM'}` dictionary, useful for lookup without touching the per-month files.

```python
from mcndroid_monthwise_split_and_loader import build_hash_to_month_map

hash_to_month = build_hash_to_month_map("final_hash_date_label_family.csv")
print(hash_to_month["abc123..."])  # e.g. '2019-05'
```

**`split_loaded_arrays_monthwise(X, y, hashes, hash_to_month)`** — splits already-loaded NumPy arrays `(X, y, hashes)` into a `{'YYYY-MM': (X_month, y_month, hashes_month)}` dictionary.

```python
from mcndroid_monthwise_split_and_loader import (
    build_hash_to_month_map,
    split_loaded_arrays_monthwise,
)

hash_to_month = build_hash_to_month_map("final_hash_date_label_family.csv")
month_buckets = split_loaded_arrays_monthwise(X, y, hashes, hash_to_month)

for month, (Xm, ym, hm) in month_buckets.items():
    print(month, Xm.shape, ym.sum())
```

**`flatten_yearwise_loader_to_months(yearly_loader_fn, years, split, csv_path)`** — wraps an existing year-wise loader and returns a merged month-wise dictionary in one call.

```python
from mcndroid_monthwise_split_and_loader import flatten_yearwise_loader_to_months
from mcndroid_cade_budget_adaptation_monthwise import load_static

months = flatten_yearwise_loader_to_months(
    yearly_loader_fn=load_static,
    years=[2013, 2014, 2016, 2017, 2018],
    split="train",
    csv_path="final_hash_date_label_family.csv",
)
```

## Step 2 — Run CADE Adaptation

Once the monthwise CSV files exist, run the CADE training and adaptation script. All four modalities use the same script with different `--modality` flags. Each modality is run **3 times** with different random seeds (`42`, `123`, `999`) to produce averaged results.

> **Tip:** If you don't have a GPU, replace `--device cuda:0` with `--device cpu`. Training will be significantly slower.

---

### Static Modality

**Run 1 — seed 42:**
```bash
python3 -u mcndroid_cade_budget_adaptation_monthwise.py \
  --modality static \
  --budgets 50,100,200,400 \
  --device cuda:0 \
  --seed 42 \
  --run_id run1 \
  --min_epochs 150 \
  --max_epochs 250 \
  --patience 10 \
  --min_delta 1e-4
```

## Output Structure

After all three steps complete, the directory looks like:

```
monthwise_data/
├── data/
│   └── monthwise_csv/
│       ├── 2013/
│       │   ├── 2013-01.csv         ← generated by Step 1
│       │   └── ...
│       ├── 2014/ ...
│       └── month_summary.csv
├── logs/
│   ├── graph_<jobid>_0.out         ← SLURM stdout (or redirect manually)
│   └── graph_<jobid>_0.err
└── results/                        ← CADE results (Step 2)
    ├── static/
    │   ├── budget50/
    │   │   ├── run1/static_budget50_seed42.json
    │   │   ├── run2/static_budget50_seed123.json
    │   │   └── run3/static_budget50_seed999.json
    │   ├── budget100/ ...
    │   ├── budget200/ ...
    │   └── budget400/ ...
    ├── graph/  (same structure)
    ├── dynamic/  (same structure)
    └── all/  (same structure)

Chen/combined/results/              ← Chen results (Step 3)
├── static/
│   ├── budget50/
│   │   ├── run1/chen_static_budget50_seed42.json
│   │   ├── run2/chen_static_budget50_seed123.json
│   │   └── run3/chen_static_budget50_seed999.json
│   ├── budget100/ ...
│   ├── budget200/ ...
│   └── budget400/ ...
├── graph/  (same structure)
├── dynamic/  (same structure)
└── all/  (same structure)
```

Each **CADE result JSON** contains:
- `per_month` — per-month metrics: F1, FNR, FPR, ROC-AUC, drift percentage, and sample counts
- `iid` / `near` / `far` — averaged metrics over each temporal regime
- Metadata: `modality`, `budget`, `seed`, `run_id`, `selection_policy`, `train_year`

Each **Chen result JSON** contains the same structure, plus:
- `method` — always `"chen_al_2023"` for easy identification when comparing results
